# Where the AI lives on the Noesis consensus layer, and why the cost objection mostly dissolves

The premise we share is that a blockchain can do more than order transactions: it can price contribution. In Noesis, a value function v(S) scores what a set of contributions is worth, that score becomes Proof-of-Mind standing, and standing is voting weight over which blocks finalize. So the "intelligence of consensus" is not a slogan. The consensus literally runs on a learned measure of contribution value. The reasonable worry you raised is cost: if intelligence sits on the hot path, every block pays for inference, and that does not scale.

The short version of our answer is that the expensive part of the AI is not on the hot path, and it is kept off the hot path by a hard constraint, not by an optimization we could later regret. Once you see the constraint, the cost objection stops being a tug-of-war between benefit and expense and becomes an ordinary train/inference-partition problem that has a clean answer.

## 1. The determinism wall decides where the model can live

Consensus requires bit-identical replication: every honest node must compute the exact same value from the same inputs, or there is no agreement to finalize. Neural inference does not satisfy this. Float reduction order, GPU nondeterminism, and library-version drift mean two honest nodes running the same weights on the same input can disagree in the low bits. So a learned model cannot run inside the hard state transition. This is not a limitation we are working around. It is the structural fact that shapes the whole design.

Concretely, the seam where value enters consensus is a `ValueOracle` interface (`node/src/lib.rs`) with a hard contract: pure, deterministic, integer, replica-identical output. The shipped scorer behind it today is a fixed-point novelty heuristic, `NoveltyOracleV0`, which satisfies that contract and is genuinely on-chain (built, and honestly labeled designed-not-learned). A neural v(S) fails the contract by construction. That failure is the reason the architecture is shaped the way it is, rather than a bug to fix.

## 2. So the model is a node-level oracle, and the chain is its harness

If the model cannot live inside consensus, where does it live? It lives in the nodes, as an oracle, not in the state transition. A node runs its model off-chain (it can be a small local model, or a heuristic, whatever it chooses) and emits pairwise judgments of the form "is contribution A more valuable than B?" These fuzzy, non-deterministic outputs reach agreement through CRPC, Commit-Reveal Pairwise Comparison, the protocol specified by Tim Cotten (his original write-up 404'd and survives only via a Wayback archive, which is why the credit is preserved here). CRPC runs over the naturally-sharded provenance graph: each cell or provenance sub-graph is an independently verifiable shard, and pairwise scores aggregate into a soft ordering per shard.

The move that will be familiar to you: the chain is a harness around fallible models, and the harness carries the difficulty. Commit-reveal defeats copying, stake and slashing defeat dishonest reporting, and pairwise-relative scoring converts an intractable absolute question ("is this contribution true or good?") into a tractable relative one ("is A better than B?"). This is the same fortress-around-the-model pattern that lets a small model match a frontier one when the harness does the heavy lifting (the result our design leans on is arXiv 2607.08938, "Better Harnesses, Smaller Models"), lifted from a single agent up to the protocol layer. The AI is fallible and non-deterministic; the structure around it is what makes the output trustworthy. Status: the oracle seam and the deterministic v0 are built; the CRPC node-oracle layer is designed/sketched, not built.

## 3. The cost objection, taken apart the way you would take apart an inference bill

The frame that makes this precise is essential versus accidental complexity. The intelligence of consensus is essential: it emerges from one small kernel, value as novel realized flow along provenance, and the rest is that kernel expressed. The expensive-model cost is accidental, sprawl we design out, not a property of the intelligence itself. With that split in mind, you are right that not all of the "AI" is expensive LLM calls, and it is worth being precise about which parts cost what, because the partition is the whole answer.

The consensus-critical computation is cheap by the determinism contract itself. Because every consensus value must be re-derivable on-VM (the reference runs on RISC-V / CKB-VM, and every security-critical input is re-derived and rejected if it cannot be reconstructed), the on-path computation is restricted to things that run in that environment: integer novelty scoring, the v5 through v8 value stack (v8 is a Bradley-Terry model over coalition features, i.e. a pairwise-preference aggregation, not a generative call), and the anti-gaming defenses, which are linear-algebra objects (a Helmholtz-Hodge decomposition of the attribution graph, whose residual circulation flags collusive rings in the same pass that prices the flow, plus within/cross/joint damping and a near-duplicate similarity floor). None of that is inference in the expensive sense. It is statistics and graph algebra with a fixed, small per-block cost, and the defense is not a separate state machine: the same computation that prices a contribution detects the gaming. That collapse of primary-job and defense into one computation is why it stays cheap, and it is the elegance argument and the cost argument turning out to be the same argument.

The pairwise comparisons that the node-oracles emit are single cheap local evaluations, and they are sharded. A node scoring "A versus B" over its neighborhood of the provenance graph is not a global per-block LLM call; it is one small model call, or even a heuristic, local to a shard, and the aggregate is what settles. Cost scales with the contribution graph and its sharding, not with a global inference per block.

The genuinely expensive computation, training a strong learned v(S), or running a richer judge, is amortized and off the critical path. It runs offline, produces parameters or labels, and those enter the hard layer only as a committed value that is then cheaply verified. This is the ordinary train-once / evaluate-many split, and it means the marginal cost of a block does not include the cost of the intelligence that produced the model.

Two things make this partition robust rather than a promise. First, the oracle is swappable: the honest cheap heuristic runs today, and a smarter model drops into the same deterministic slot later with no rebuild (we proved the swap is real with a byte-identical no-op test and a genuine behavior-changing test). Cost is therefore a dial, not a fixed tax. Second, the endgame removes even the re-execution question: zkML. Prove that a committed model (a weights hash) produced score X on input Y, and the chain verifies the succinct proof without re-running the model, using the same zkVM machinery as our Phase-3 work (RISC Zero). That yields a deterministically verifiable output from a model that is non-deterministic to run, which is the clean resolution of the whole determinism wall. Status: zkML is open (env- and spec-gated); the optimistic mode (commit-reveal + stake + dispute/slash) is the near-term path and reuses the equivocation slash-before-count already wired onto the live finality path.

So the honest form of "benefits outpace cost" is narrower and more defensible: the intelligence that is expensive is amortized offline, the intelligence on the hot path is cheap deterministic aggregation, and the bridge between them is either a disputable committed value or a zk proof. The expensive model is the thing we design out of the hot path, not the thing we pay for per block.

## 4. Why the intelligence is load-bearing, not decorative

It would be fair to ask whether the value measure earns its place, given the trouble of keeping it off the hot path. It does, because it is the anti-plutocracy franchise. The consensus mix weights Proof-of-Mind at the majority of the vote (pow 0.10 / pos 0.30 / pom 0.60, `node/src/lib.rs`), and finality runs on PoS and PoM only, with PoW excluded because it is reorgeable and belongs to production and the money layer, not safety (`node/src/runtime.rs`). The property that matters is the anti-concentration floor: each dimension, stake and contribution, must independently supply at least 50% of its own dimension to finalize (`MIN_DIM_BPS`, `runtime.rs`). Capital that holds all the stake still controls only the PoS dimension; the contribution dimension must independently clear its floor. Capital cannot finalize without contribution's consent. That is the whole thesis, and it only exists if there is a real, hard-to-game measure of contribution. The learned v(S) is what makes the contribution axis mean something; the floor is what makes a gamed model insufficient on its own. Neither a captured model nor pure capital finalizes alone.

## 5. The endpoint: aligned agents are the nodes, and the nodes are the chain

Follow the node-oracle idea to its conclusion. The thing that runs a model and emits a pairwise judgment is an agent. An AI-aligned agent running as a node is a validator, and since the nodes are what constitute the chain, aligned agents become the chain. Consensus is then a mesh of aligned agents reaching graded agreement over the provenance graph, with no capturable center. The CRPC sketch names this as the direction the axis points: the same opt-in graded-agreement layer is a candidate substrate for an agentic mesh, many agents coordinating on soft truth without a center. Status: this is the design direction, not a built system, and we label it that way.

This reframes the cost question one last time, and it is the reframe we think matters most to you. The inference is not a tax the protocol pays on top of consensus. It is the participation cost an agent already bears to be a validator, the way a proof-of-stake validator bears the cost of running a node. When the agents are the nodes, running the model is not an extra line item; it is what being a node means. The reference agent we run, Jarvis, is deliberately kept on free-tier models as an anti-capture stance, so the canonical node is cheap by design rather than by accident, and no single provider owns the substrate.

The obvious worry, that agents-become-the-chain is a centralization risk, is answered by the same structure that bounds humans. The franchise is contribution, not capital: an agent gains standing by producing value that the measure and its peers score highly, not by holding tokens. The anti-concentration floor then prevents any single agent, cluster, or pool of capital from finalizing alone. So aligned agents constituting the chain is not a new capture surface; it is the separation-of-powers applied to a validator set made of minds rather than only of stake. This is the point where Noesis stops being a chain that uses AI and becomes a substrate where aligned intelligence is the consensus.

## 6. Honest status, because the honesty is the moat

We hold a strict built / designed / open line and never round up, so you can calibrate what is real.

Built (reference implementation, tested): the deterministic ValueOracle seam and the v0 novelty scorer; the v5 through v8 value stack including the Bradley-Terry outcome gate; the anti-concentration floor and the consensus/finality mixes; equivocation slashing that strips a double-signer's weight before any weight is counted on the finality path.

Designed, not built: the CRPC node-oracle layer (Tim Cotten's commit-reveal pairwise comparison as the graded-agreement engine over the shard graph); the swap of a learned model into the oracle slot.

Open, and these are the ones we would most value a reader's eyes on: the non-interference proof, that a compromised soft layer can degrade coordination but can never mint value or finalize a bad cell, which is the load-bearing safety property before any of the soft layer ships; the learned v(S) on real labels, which is data-gated rather than effort-gated (on real Deep Funding data a rich-feature judge generalizes at about 0.60 on a repo-disjoint split, a modest, popularity-driven real signal, which is honest-label prediction and explicitly not the adversarial un-gameability, which rests on the structural defenses); and zkML as the trustless endgame for the oracle output.

## 7. Why this is the right conversation to have with AI researchers

The hard parts here are ML-systems questions wearing consensus clothes: where inference is allowed to be non-deterministic, how to turn a fallible model into a trustworthy claim with a harness, how to partition amortized training from per-block evaluation, and how zk verification collapses the two. The three open items above are exactly where an AI researcher would see something a protocol designer alone might miss, especially the non-interference boundary and the learned v(S). If any of this is worth a longer exchange, we would rather show the code than the pitch.

The frame above this doc, rather than beneath it, is written up as an essay: *Essential Complexity: The Organism and the Machine* (`docs/research/essential-complexity-organism-and-machine.md`; published at https://medium.com/@tiptaptangsun/essential-complexity-the-organism-and-the-machine-89b85fc50087). It argues that elegance is generative parsimony rather than minimality, that a system is an organism when its complexity unfolds from a small kernel instead of accumulating as bolted-on parts, and it uses Noesis as its own honest test case, including the part where the learned value measure returned NULL on its first real-data test against Ethereum's Deep Funding jury labels and we declined to relocate the claim afterward. The one line from it most relevant here: once you are measuring value against strategic reporters, the only un-gameable measure is one that adapts to the gaming. That is Goodhart and reward-model overoptimization restated as a design constraint, and it is why the measure has to be learned and adaptive rather than a fixed rule, and why we treat un-gameability as essential complexity to be earned rather than a trick to be chosen. The essay holds the same line this doc does: real is not proven, and the discipline is to build toward it without rounding up.
