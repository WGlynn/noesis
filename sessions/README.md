# Session summaries

Human-legible, plain-English recaps of each working session on Noesis — one file per session,
named `YYYY-MM-DD-topic.md`, newest by date.

**Why this folder exists.** The internal handoff files (`internal/CONTINUE.md`, `ROADMAP.md`) are
dense and operator-facing — great for resuming work, hard to read day-to-day. These summaries are the
opposite: they explain *what happened and why it matters* in language you can read over coffee, so the
progress is easy to internalize even when a lot ships in one session. Written for Will first, and for
any collaborator who wants the story without reading the diffs.

**What each summary covers:** what shipped (in human terms, not just commit hashes), why it matters to
the thesis, the honest status (built vs designed vs open), and what's next. Numbers and commits are
included but never the whole point.
