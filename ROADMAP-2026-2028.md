# Noesis Roadmap 2026–2028

> **The shift (2026-07-02).** Patent 1 is filed — a snapshot of the invention on that date, not
> a living document. The real work begins: build the protocol. The patent was a milestone; the
> protocol is the destination.
>
> **North star (the one obsession):** *Can someone reproduce Noesis from the patent and the
> documentation alone? When the answer is "Yes," we have done something unusual.*
>
> **Goal:** make Noesis so technically compelling that people want to implement it regardless of
> whether they have ever read the patent. Then the patent becomes what it should be — a legal
> foundation under a technology people genuinely want to adopt.
>
> **Operating mindset (Rodney's AI, 2026-07-02):** stop asking *"how do I prove Noesis is right?"*
> Ask *"how do I make it easy for someone else to prove me wrong?"* If they can't, confidence
> grows; if they can, you learned it before mainnet. This produces stronger systems.
>
> **The milestone that changes valuation:** not another patent. It is the first time someone
> *other than us* says "we built on / tested / want to integrate Noesis." Today's honest status:
> an investable *concept*, not yet an investable *business*. No fake dollar number.

problem · what are the invariants · what trade-offs · where should it NOT be used · how are future
protocol changes evaluated. Becomes the philosophical foundation for docs, code review,
governance, and future patents. Draft: `docs/MANIFESTO.md`.

## Phase 2 — Build (full-time intellectual effort)

**Priority 1 — make the reference implementation impeccable.**
- It is the canonical implementation. The docs must reflect the architecture exactly.
- Every mechanism deterministic. Every patent claim must hold against the code.
- Drive every 🟡 designed → ✅ built; keep 🔬 open items honestly marked. No rounding up.

**Priority 2 — build trust via radical transparency (public roadmap):**
- Patent Pending ✅ · Reference Implementation ✅ · Interactive Demo ✅
- Documentation 🚧 · SDK 🚧 · Developer Testnet 🚧 · External Review 🚧 · Performance Benchmarks 🚧

**Priority 3 — write the whitepaper (NOT the patent).**
- A proper ~30-page engineering paper. Diagrams · attack analysis · economics · design rationale
  · implementation notes.
- Division of labour: **the patent explains WHAT; the whitepaper explains WHY.**

**Priority 4 — publish under a deliberately-chosen licence.**
- Code = copyright; the architecture is patent pending. Complementary protections. Choose the
  licence carefully (defensive-open posture, consistent with the open ethos).

## Phase 3 — Patent 2 (not now)

  actually a separate invention," log it there. Collect inventions; claim later.

## Phase 4 — Credibility via what patents can't do
- Benchmarks · simulations · formal proofs · write-ups · independent review.
